library not found for - XXX 的解决方法

发表于 2016-01-23   |   分类于 遇到的trouble

出现问题的操作

使用Cornerstone管理仓库，把最新的项目提交到远程仓库，在另一台电脑上检出项目后编译出现“library not found - lssl”

原因分析

首先字面意思就是说 xxx库文件找不到了。由于是我们在项目中使用了一些第三方的库，当Xcode在编译的时候却找不到这个库，从而导致如此。

找不到文件分两种情况：

  • 项目中没有添加这个库文件
  • 这个库文件有一个绝对路径（Xcode查找文件只是在工程内部查找，所以要重新设置该文件的路径）

解决办法

Xcode找不到库文件，那么我们就要让Xcode知道这个库文件在哪里，从而在编译的时候轻松的找到它。

添加静态库路径

在工程的Target中选中要执行编译的某个target,打开Build Setting，在“Library Search Path”中添加缺失链接库的所在文件夹的路径即可

设置SVN(Cornerstone)
