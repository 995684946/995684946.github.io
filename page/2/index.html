UITableView下移问题

发表于 2015-10-04   |   分类于 iOS

在做项目中遇到在我们继承自UIViewController的View上添加一个TableView，这个界面需要判断用户是否登录决定展示数据还是登录跳转界面，结果在登录后再次展示界面的时候整个tableView的frame出现了下移。再次切换界面后有回复正常。当时就头大了……

原因：

  • 默认情况下, 如果一个控制器A处在导航控制器管理中, 并且控制器A的第一个子控件是UIScrollView, 那么就会自动调整这个UIScrollView的contentInset
    • UIEdgeInsetsMake(64, 0, 0, 0) // 有导航栏
    • UIEdgeInsetsMake(20, 0, 0, 0) // 没有导航栏
  • 默认情况下, 如果一个控制器A处在导航控制器管理中, 并且导航控制器又处在UITabBarController管理中, 并且控制器A的第一个子控件是UIScrollView, 那么就会自动调整这个UIScrollView的contentInset
    • UIEdgeInsetsMake(64, 0, 49, 0)

解决办法：

iOS7在Conttoller中新增了这个属性：

automaticallyAdjustsScrollViewInsets，当设置为YES时（默认YES），如果视图里面存在唯一一个UIScrollView或其子类View，那么它会自动设置相应的内边距，这样可以让scroll占据整个视图，又不会让导航栏遮盖。

我们设置automaticallyAdjustsScrollViewInsets这个属性为no，就可以解决这个问题。然后我们自己设置tableview的frame，将状态栏和导航栏的位置空出来就可以了。

自定义调试模式下Log输出

发表于 2015-07-30   |   分类于 iOS

自定义Log

NSLog非常耗性能,如果程序发布了,还一大堆NSLog,用户用起来会很卡。 解决方式:自己写个Log,做处理,在发布的时候不打印,调试的时候才打印。

#ifdef DEBUG

#define LSBaseLog(...) NSLog(__VA_ARGS__)
#define LSFuncLog LSLog(@"--%d,%s",__LINE__,__func__)
#define LSLog(fmt,...) NSLog((@" \n%s\n -line %d \n -" fmt),__PRETTY_FUNCTION__, __LINE__, ##__VA_ARGS__)

#else

#define LSBaseLog(...)
#define LSFuncLog
#define LSLog(fmt,...)

#endif
